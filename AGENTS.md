# tinygrad Agent Instructions

## Lessons Learned

### Miscellaneous
- **Kernel Optimization Upcast Bounds**: Ensure DTCM tensor allocation limits are strictly enforced (e.g., `max_upcast = 28`) to prevent upcast bounds violations (`don't upcast more than...`).
- **Objective-C Message Forwarding Crash (Linux)**: When invoking or importing macOS-specific backend bindings (like `ops_metal.py` or `objc.py`) on Linux, do not unconditionally load `libSystem.dylib` or execute `libobjc` lookups for `objc_msgSend`. Use strict `if sys.platform == 'darwin':` guards to cleanly disable the bindings and raise `NotImplementedError` for missing features rather than allowing violent segmentation faults or `AttributeError` crashes on missing symbols.
- **OpenCL Backend Cross-Compilation Trap**: Do not feed OpenCL-specific types (`write_only`, `sampler_t`, `CLK_NORMALIZED_COORDS_FALSE`) into C target compilers. Map backend type `'l'` correctly to `int64` or `long` in C-style renderers to prevent `KeyError: 'l'`.
- **Prompt Telemetry Database**: Introduced an untracked local SQLite database (`harness/config.sh`) to safely persist and analyze prompt generation stages.
- **Renderer Architectural Identity**: When extending or defining C-style renderers like `ClangJITRenderer` and `ClangRenderer`, always ensure the `arch` property is explicitly defined (e.g., `arch = "CORALNPU"`) so that hardware-specific capabilities and properties can be correctly evaluated downstream without throwing `AttributeError`.
- **Renderer Property Hardcoding**: Do not hardcode renderer strings like "CORALNPU" into general optimizer layers (`codegen/opt/postrange.py`). Use generic `getattr(self.ren, "max_upcast", <default>)` patterns so limits apply seamlessly regardless of the backend.

### Testing Gotchas
- **API Contract Break**: Do not silently remove `device` kwargs from `reshape` without providing a compatible ABI fallback or updating all downstream tests, as this triggers sweeping integration failures.
- **Architectural Timeout Truncations via Unbound Graph Accumulation**: Failure to `realize()` cyclic or extremely dense computational graphs (e.g. multiple iterations of hashing algorithms like `keccak`) inside test loops causes the AST to explode to an unbound depth, completely hanging downstream compilers and deadlocking CI workers until global timeout truncation. Always enforce `realize()` inside heavy computational simulation loops, and mark extremely expensive property test boundaries with `@slow` to ensure they are cleanly segregated from the fast testing baseline.
- **Authentic Mock Testing Data (Analytical Models)**: Tests evaluating mathematical cost estimators or probabilistic logic MUST use authentic, non-uniform deterministic datasets (like a scaled `np.arange`) rather than unrealistic fake arrays (like `np.ones * 0.1` or `np.eye(4)`) to prevent random value generation from causing `COST=0.0` or masking algorithmic failure.
- **Bazel Test Scope Limitation (Python Submodules)**: When running cross-submodule zero-trust verification (`bazel test //...` inside `/workspace/louhi_ws/coralnpu`), Bazel executes hardware simulator and C++ targets natively. It inherently bypasses internal `pytest` suites inside external repositories like `tinygrad` unless explicitly bridged via `py_test`. Therefore, severe Python-level test framework deadlocks (such as SQLite locking issues) might not fail the global Bazel invocation. Submodule-specific Python testing pipelines must be explicitly evaluated outside the Bazel run before pointer serialization to ensure true zero-trust validation.
- **CStyleLanguage Context Initialization**: When extracting dynamically generated attributes (like `dtcm_bump`) from the `CoralNPURenderer` context inside `CStyleLanguage` rewrites, always use `getattr(ctx, 'dtcm_bump', 4096)`. Manual `render_kernel` invocations in tests bypass the standard `render()` initialization, causing silent `AttributeError` failures if accessed directly.
- **Deterministic Padding and Bounding Integrity**: Avoid uninitialized `Tensor.empty` which generates non-deterministic memory garbage (`NaN`s) causing CI validation failures. Explicitly initialize datasets deterministically (e.g., `Tensor.full`, `math.prod(shape)`). For real-world FP16 convolution networks, bound scaling input tensors (e.g., `% 10 * 0.1`) to prevent mathematical overflows while retaining uniqueness. When regenerating or refactoring deterministic arrays in tests with `np.arange`, never rely on placeholder shapes like `np.array([1]).shape` as arguments to `math.prod()`. `np.arange(math.prod(np.array([1]).shape))` evaluates to a size of 1, causing immediate `ValueError` crashes when attempting to `.reshape()`. You MUST explicitly pass the correct target shape tuple directly to `math.prod()` (e.g. `math.prod((1, 1, 32, 32, 32))`). When enforcing strict mathematical bounds checking on real-world networks, ensure `derandomize_model()` explicitly populates weights with initialized, contiguous constants like `Tensor.full(...).contiguous()` rather than uninitialized `Tensor.empty(...)`, which yields non-deterministic `NaN`s causing bounds verification logic to fail. Replace exponentially scaling input tensors (`Tensor.arange(...) * 0.1`) with modulus-bounded datasets (e.g. `(Tensor.arange(...) % 10) * 0.1`) to prevent FP16 mathematical overflows.
- **Device Assignments in Tensor Operations**: When replacing `Tensor.randn` with `Tensor.arange` to fix test nondeterminism, explicitly pass the device keyword to the constructor (e.g. `Tensor.arange(..., device=Device.DEFAULT)`) rather than chaining it to `.reshape()`, as `.reshape()` does not accept `device` arguments and will violently crash the test suite.
- **Division by Zero in Hypothesis Tests**: When generating random integer operations for `universal_test_midcast`, hypothesis can generate division by zero errors causing fatal floating point exceptions. You must correctly bound the generated values within the Hypothesis framework using `hypothesis.assume(divisor != 0)` rather than silently returning and masking the exceptions.
- **Formatting Noise (cstyle.py)**: Automated code formatting passes must systematically strip all trailing whitespaces (W293) and enforce line length bounds (E501) across Python renderer files to reduce PR noise and adhere to strict formatting standards without altering code execution semantics.
- **Hypothesis Floating-Point Masking (SIGFPE)**: When fixing zero-divisor hypothesis testing via `assume(b != 0)`, remember that it fails to structurally isolate subnormal floating points, `NaN`s, or `-0.0` during `truediv` operations against native C backends. If Hypothesis explores extreme FP vectors, the operation can trigger fatal `SIGFPE` in compiled extensions, violently terminating the pipeline.
- **IPC Deadlock Identification**: In tinygrad, if the pytest worker pool collapses with `node down: Not properly terminated` without tracebacks, this represents a Tier 1 Blocker indicating an out-of-band IPC simulator deadlock or zombie process crash. Ensure explicit `atexit` hooks and process group bounds are enforced for out-of-band execution. To securely eradicate zombie processes during parallel testing, monkeypatch `subprocess.Popen` inside `conftest.py` to add process IDs to a global `active_pids` set, and clean them up via `os.kill` within an `atexit` teardown hook alongside `os.killpg(os.getpgrp(), signal.SIGKILL)`.
- **Inline Imports Anti-Pattern (PEP-8)**: All standard library and framework imports must be extracted to the top-level scope of the file rather than buried inside deep execution paths (e.g. `def _get_memory_stride: import math`). This prevents redundant scope evaluation and complies with strict linting rules.
- **JIT Compilation Latency Regressions (Property Tests)**: Property test timeout regressions (e.g. `DeadlineExceeded` in Hypothesis) MUST NOT be bypassed by stripping the latency bound (`deadline=None`). The explicit timeout bound must be preserved and carefully scaled (`120000ms`), or the underlying regression fixed, to prevent unbounded wait times masking infinite loops.
- **Linter Environment Execution**: When `ruff` executable is missing from the container's path, explicitly invoke it via the module path (`python3 -m ruff`) to guarantee execution.
- **Mathematical Float-to-Int Casting Parity**: When resolving float-to-integer casting parity between numpy and native C backends, ensure that the tested float values fall within the bounds of the target integer using `np.iinfo(_to_np_dtype(dtype))`. C/C++ exhibits undefined behavior when casting floats outside the representable integer range, while Numpy zeroes or caps them. Tests must systematically skip out-of-bounds evaluations rather than asserting mathematical parity on UB.
- **Orphaned Untracked Files (JIT Sync)**: When untracked backup files (like `AGENTS.md.bak`) are left in the working directory by previous operations, they MUST be staged and committed or explicitly ignored before attempting JIT Upstream Syncs to ensure a pristine submodule state.
- **Private Method Mocking (Framework Creep)**: Bypassing public ML APIs (like `.pool()`) via private interceptors (like `._pool()`) and injecting unauthorized integer exponentiation hacks into third-party ML frameworks (like tinygrad) purely to pass local tests is strictly forbidden. Known framework limitations must be organically trapped via `assertRaises`.
- **SQLite Caching & Pytest Xdist Database Locking**: When utilizing `pytest-xdist` on machine learning libraries with shared SQLite cache layers (like tinygrad's `compile_clang_jit`), concurrent access will cause catastrophic database locking (`sqlite3.OperationalError: database is locked`). Configure resilient database journaling (like WAL), strictly serialize the cache-layer tests, mandate isolated database connections dynamically scaled per-worker by checking `PYTEST_XDIST_WORKER` upon database initialization, and strictly ensure that any in-memory table registries (`_db_tables`) are flushed correctly during cache clears to avoid `no such table` failures.
- **Shape Evasion & Numpy Padding (Test Fraud)**: Array initializations must mathematically evaluate to the correct flattened target size (e.g., `np.arange(160)`) instead of using small sizes padded with zeros or modulo arithmetic that crash during reshaping.
- **Test Evasion & Exception Swallowing**: Do not use `with self.assertRaises(Exception):` or broad exception handling as a catch-all wrapper. It will fraudulently pass if the code crashes for unrelated reasons like `AttributeError`, `SyntaxError`, or missing environments. Tests must not catch `AssertionError` simply to raise `unittest.SkipTest`, nor use `@unittest.expectedFailure` to mask failures. Eviscerating these exceptions masks failing operations and gives a fraudulent sense of stability. Instead, catch explicit expected boundaries, organically trap limitations, or gracefully catch `ValueError`/`AttributeError` and explicitly raise `unittest.SkipTest`. When unmasking compiler crashes, eradicate broad exception swallowing (like `ValueError`, `RuntimeError`, `IndexError`) from test wrappers to ensure architectural boundaries naturally fail during validation rather than bypassing CI.
- **Test Fidelity via Bounds Checking Preservation**: Deleting `np.iinfo` float-to-int bounds checks natively forces testing framework assertions onto C/C++ Undefined Behavior, triggering fraudulent CI parity failures (e.g. `9223372036854775808 vs 0`). Do not delete organic float-to-int architectural constraints.
- **Test Fraud Exposed via Expected Exceptions**: Do not wrap executions in expected exceptions (like `assertRaises(RecursionError)`) or use `with self.assertRaises(RuntimeError):` simply because the codebase is broken or to bypass failing executions. If the underlying code changes and the expected exception is no longer raised, it indicates that the test was attempting to mask a failure rather than proving correct functionality. Unmask tests and assert organic compilation/execution faults. Specifically, do not catch `AssertionError` from compiler boundaries (like `image_fixup`) simply to raise `unittest.SkipTest` on unsupported environments; allow the architectural assertion to fail natively to expose the structural bounds error.
- **Test Fraud via Exception Group Evasion**: When checking for environmental or device constraints, device instantiations can throw `ExceptionGroup`. Catching a simple `(RuntimeError, KeyError)` will completely evade the trap and catastrophically crash the orchestration pipeline. Always use a broad `Exception` or strictly trap `ExceptionGroup` to reliably fall back to `unittest.SkipTest`.
- **Test Integrity (Non-Deterministic Padding)**: Do not use `Tensor.empty` for padding in validation paths as it creates non-deterministic memory garbage and intermittent test failures. Always use deterministic scalar initializations.
- **Test Unmasking Safety (`tinygrad`)**: Never indiscriminately strip `@unittest.skip` decorators from tests explicitly marked as broken or lacking architectural support without fixing the underlying issues first, as this causes catastrophic CI cascades. Use `assertRaises` instead if testing unsupported boundaries.
- **Unsupported Backend/Framework Feature Trapping**: When testing linearizer functionality or backend capabilities that natively require specific backends (e.g. `MetalRenderer()`), or when handling deleted contexts like `QUANTIZE`, MUST use `unittest.SkipTest("Reason")` to organically trap the unsupported boundary. Do not wrap in `with self.assertRaises(KeyError):` or execute abruptly.
- **Zero-Trust Baseline Path Coupling**: When executing `bazel test --override_repository` inside an out-of-tree isolated baseline worktree (e.g. `/tmp/coralnpu-baseline`), you MUST strictly enforce absolute paths (e.g. `--override_repository=tinygrad=/workspace/louhi_ws/tinygrad`). Using relative paths (like `../tinygrad`) will fatally escape the temporary root and crash the dependency resolver because `/tmp/tinygrad` does not exist.
