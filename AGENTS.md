# Tinygrad Agent Instructions

## Lessons Learned

### Architecture & Integration
- **Architecture Update:** Added a dynamic `coralnpu` backend compiling UOps into GCC RISC-V Zve32x code. Simulates out-of-band execution using an embedded neural network cost model (`BEAM_COST`) for auto-tuning.
- **Submodule Pointer Synchronization:** Subsequent commits made to the submodule (e.g., adding localized lessons to this file) MUST be immediately followed by serializing the new submodule pointer in the parent workspace to prevent state drift.
- **Memory Abstraction:** Implement a Virtual Memory Management (VMM) abstraction or pseudo-driver layer in `ops_coralnpu.py` to handle ITCM/DTCM mapping internally, preventing hardware abstraction leaks.
- **PyBind11 Isolation:** Wrap PyBind11 calls in a managed `multiprocessing` pool.
- **IPC Watchdogs & Heartbeats:** Implement strict Heartbeat & Timeout Watchdogs differentiating compile vs runtime waits. 
- **IPC Watchdog Testing & Evasion:** Tests should validate `multiprocessing.shared_memory.SharedMemory` channels and strictly assert that `multiprocessing.Process.join(timeout=...)` successfully terminates a hanging process and raises the expected timeout failure. Simply asserting `RuntimeError("Cannot execute RISC-V binaries natively")` provides zero test coverage for the out-of-band simulator IPC watchdog; mechanics must be verified by passing execution to an actual out-of-band dummy subprocess.

### Git & Environment Management
- **Upstream Rebase Safety:** Do not apply new logic or test fixes while actively resolving a rebase conflict. The rebase operation must strictly resolve conflicts and complete (`git rebase --continue`) before subsequent test suite remediations are atomically applied.
- **Upstream Validation:** Use `git worktree add ../tinygrad-baseline upstream/master` to test upstream state without altering the current tracking branches or `.git` index, completely isolating tests from the local workspace.

### Testing & Verification
- **CPU-Only Backend Enforcement:** To prevent ROCm/GPU compiler crashes in the orchestrator container, tests must forcefully use the CPU backend and ignore hardware-specific folders using `CPU=1 pytest --timeout=120 --timeout-method=thread test/backend/ test/unit/ test/null/ test/test_tiny.py`.
- **Test Dependencies:** Ensure `huggingface_hub`, `pytest-xdist`, and `pytest-timeout` are installed (`pip install huggingface_hub pytest-xdist pytest-timeout`) as they are required dependencies for tinygrad tests, preventing spurious failures during cross-submodule integration.
- **Deadlock Identification via Truncation:** When a Python test suite abruptly truncates mid-execution and is killed by the global system without dumping a timeout stack trace, it strongly indicates a fatal IPC/RPC out-of-band simulator deadlock.
- **IPC Reliability & Timeouts:** Prevent silent timeout deadlocks by adding aggressive timeout fallbacks for out-of-band IPC executions. Use `PYTHONUNBUFFERED=1 pytest -n auto --timeout=120 --timeout-method=thread` to force thread dumps on timeout. *Crucially, parallel execution (`-n auto`) is strictly required to prevent the overarching harness from blindly terminating the process before pytest can dump threads.*
- **Fatal x86/RISC-V `ctypes` Host Loading:** Do not attempt to use `ctypes.CDLL` to load cross-compiled `.so` shared libraries built for RISC-V (`-march=rv32imf_zve32x`) into the native x86 Python host process. This triggers `OSError: Exec format error`. The backend must explicitly dispatch execution to an out-of-band hardware simulator (e.g. `mpact` or `oss-cosim`).
- **Destructive Test Remediation & Erosion:** Do not simplify symbolic math parameters or mathematical constants (like removing the `4*c` factor in `test_uop_symbolic.py`) merely to achieve a passing test. Do not strip `before` state capture mechanisms in memory tests (`test_invalid_tensor.py`). Avoid 'happy-path only' coverage and silent CI degradations. Masked tests must assert pre-operation state wasn't clobbered.
- **NaN Validation in Memory Captures:** When restoring `before` memory assertions (like `test_invalid_tensor`), avoid failing CI validations due to `NaN` comparison nuances in Python. `math.isnan` logic must be actively preserved alongside restored assertions to prevent tests from spuriously breaking during memory dump equality checks.
- **Fake Data Execution Padding:** Replacing `np.zeros` padding with deterministic Gaussian noise (`rng.randn`) does not resolve "happy-path" testing. Generating deterministic noise instead of zero-matrices still proves absolutely nothing about actual analytical logic or real weight execution. True deterministic golden weights must be used and exact mathematical outputs strictly asserted.
- **Hardware Backend Testing:** The `coralnpu` rendering and runtime backend must have dedicated unit tests inside the ML frontend (`tinygrad/test/`), rather than blindly relying on out-of-band simulator validation.
- **CoralNPURenderer Unit Testing:** When testing AST limit boundaries (e.g. `RuntimeError` due to >32 float allocations), nodes must have identical depth constraints. Chaining dependents results in scalarized depths preventing the max-width check from exceeding limits. Use `unittest.mock.patch` to sidestep strict file constraints for missing ML models during environment-agnostic tests.
- **Timeline Testing:** Ensure timeline testing (e.g., `test_sqttmap.py`) validates timestamps, execution ordering, and duration boundaries rather than purely counting events.
- **ProfileRangeEvent Validation:** When implementing timeline checks on `ProfileRangeEvent` objects from `sqtt_timeline`, use `isinstance` for filtering, explicitly assert optional fields like `e.en` are not None before comparisons, and safely extract names from `TracingKey` using `getattr(e.name, 'display_name', e.name)` to prevent runtime and type assertion errors.
- **Test Suite Granularity:** Comprehensive test creation tasks (e.g., covering operations, renderers, IPC layers) must be explicitly decomposed into distinct, atomic test creation tasks to prevent monolithic execution.

### IPC & Orchestration Bottlenecks
- **PyBind11 Out-of-Band Isolation:** Native PyBind11 bindings inside the Python tuning loop cause crashes. Extract and run calls via `multiprocessing` workers using zero-copy shared memory.
- **Strict Timeout Watchdogs:** Subprocess workers must implement a fault handler (e.g., catching explicit IPC timeouts) to dump stack traces before termination.
- **RISC-V Native Execution Error:** x86 Linux processes cannot natively execute RISC-V `ctypes`. The host-side `ctypes` bindings for RISC-V binaries must be severed and execution must route to an out-of-band hardware simulator.
- **IPC Memory Mapping:** When using `multiprocessing.shared_memory.SharedMemory`, assign `shm.buf` directly to memory views. Do not cast it via `ctypes` `from_buffer` to avoid unnecessary dependencies and type checking errors.
- **Autogen Dependencies (`libclang`):** The autogen tests in `test/null/test_autogen.py` require `libclang`. If tests crash with `AttributeError: failed to load library libclang`, ensure `LIBCLANG_PATH` is correctly set in the environment or installed globally (e.g., `/usr/lib/llvm-19/lib/libclang.so`).
- **Test Integrity & Environment Masking:** Do not artificially mock `subprocess.check_call` using `unittest.mock.patch` to force `FileNotFoundError`. Tests verifying cross-compilation failure must organically use `shutil.which` to check presence, naturally failing and asserting `RuntimeError`, or explicitly skipping if the binary is present.
- **ML Cost Validation:** Tests relying on ML cost model assertions must NOT filter by `cost > 0`. Because stochastic noise was replaced by deterministic outputs, the predicted cost for models can legitimately evaluate to exactly `0.0`. Hard filters on positive values will incorrectly drop valid model generations, failing the CI with `No valid candidates found`.
- **Submodule State Drift Prevention:** Always append an explicit submodule serialization phase to the `PLAN.md` file whenever codebase modifications (like QA and cost filter fixes) conclude without a pre-existing serialization task.
- **Bare Exceptions:** Never use bare `except:` blocks to swallow legitimate type or runtime errors during execution or AST traversal. Explicitly use strict structural type verification (e.g., `hasattr`, `isinstance`) to avoid silently swallowing legitimate programmatic errors like `AttributeError` or `KeyboardInterrupt`.
- **Fraudulent Mocking of IPC:** Never aggressively mock `subprocess.run` in `ops_coralnpu` to fake out-of-band hardware simulation. Tests must compile real C binaries and organically assert shared memory outputs or timeouts to maintain environment fidelity.
- **Zero-Trust Baseline Synchronization Safeguards:** Always mandate `git status` and `git stash` inside the submodule prior to invoking baseline extraction (`git worktree add`) or rebasing logic to definitively prevent untracked state entanglement and protect uncommitted orchestration state from accidental destruction.
- **Worktree Initialization (tinygrad):** When performing upstream validation using `git worktree add`, ensure `git worktree prune` and `rm -rf` are executed on the target `/tmp/` directory prior to initialization to prevent failures from stale paths.
- **IPC Environment Fidelity:** Never mock `subprocess.run` to fake IPC boundaries. Tests must definitively compile the real C binary using the cross-compiler and dispatch it to the out-of-band simulator.
- **DTCM Double-Buffering Overflows:** When tiling loops for hardware memory constraints (e.g., 32KB DTCM), chunks must be strictly scaled to accommodate stack overhead (e.g., <= 12KB chunks, not 16KB) to prevent corruption.
- **W8A8 Activation Quantization Void:** Before issuing an INT8 vector MAC operation (VDOT), the incoming fp16 activation tensors MUST be dynamically quantized; issuing x_int8.dot(w_int8) directly onto unquantized streams produces mathematical garbage.
- **Missing ROCm Device Library**: In environments lacking the ROCm device library, AMD handwritten tests (like `test_mfma_fp8`) must be explicitly skipped using `@unittest.skip` to prevent CI failures when testing the `amdgcn-amd-amdhsa` target.
- **Upstream Synchronization Conflict Override:** When performing `git rebase upstream/master` on local branches holding old autonomous orchestration commits, use `git checkout --ours <file>` to strictly enforce the pristine upstream baseline (`HEAD`), discarding the old feature branch states that attempt to regress previously resolved logic (e.g. `subprocess.check_call` vs `multiprocessing` wrappers). **CAUTION:** This rule applies ONLY to `rebase` contexts where `HEAD` is the upstream branch. Do NOT use `--ours` blindly during a `git stash pop` conflict, as `HEAD` is your local branch and `--ours` will permanently wipe out your uncommitted work.
