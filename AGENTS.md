# Tinygrad Lessons Learned

## Testing & Architecture
- **Optimizer Tolerance Drift:** Multiple Muon optimizer tests fail due to numeric drift. Tolerances (`rtol`, `atol`) must be evaluated or numerical implementation corrected.
- **Real World Memory Boundaries:** Memory tracking assertions must reflect the new, more efficient memory usage baseline instead of arbitrarily asserting fixed bounds limits (e.g., `test_train_mnist` and `test_bert` failing because actual memory consumed is far lower than the hardcoded threshold).
- **Scheduler/AST Assertion Bug (LAMB Offload):** The graph scheduler must properly handle constants (`Ops.CONST`) during LAMB CPU offloading instead of expecting an `AFTER` op.

### QA Directives & Testing Rules
- **Deceptive Test Unmasking:** Organically catch and handle unsupported exceptions (like `UnsupportedDevice`) structurally. Do not merely swap complex skip macros for simpler ones. Do not delete skip macros and leave empty lines; instead, address the underlying code limitations.
- **Fraudulent "Happy-Path" Padding via Fake Data**:: Tests verifying mathematical algorithms (like memory boundaries or cost models) MUST NOT be driven by random Gaussian noise (e.g., `Tensor.randn`). Random data completely masks algorithmic and mathematical failure surfaces (like accumulator overflows or zero-divisor faults) that are only exposed by deterministic, non-uniform authentic datasets. Always use deterministic datasets (e.g. a scaled `np.arange`).
- **Infrastructure Exhaustion via Unmasking Slow Tests:** The `@slow` decorator must strictly gate on `os.getenv("RUN_SLOW")`. Replacing it with a naive passthrough (`def slow(fn): return fn`) indiscriminately runs massive architectural benchmarks on standard CI, causing 120s timeout truncations and PID exhaustion.
- **Multi-GPU Masking:** The `needs_second_gpu` wrapper must use `raise unittest.SkipTest("missing second GPU")` on exception, not a silent return, to correctly log skips on single-GPU CI runners.
- **Optimizer Mathematical/Tolerance Drift**: Re-evaluate the tolerances or correct the numerical implementation in the optimizer logic when numeric drift exceeds deterministic testing tolerances.
- **Real World Memory Boundary Assertions**: Adjust the `max_memory_allowed` threshold downwards to reflect new, more efficient memory usage baselines instead of arbitrarily asserting fixed bounds limits.
- **Selective File-Level Evasion**:: When fixing localized defects within a file, agents must recursively audit and fix ALL related issues flagged in the orchestrator's ledger within that exact same file, rather than superficially fixing one and intentionally abandoning the others (e.g., fixing `@slow` while ignoring `needs_second_gpu` in the same helper file).
- **Testing Fraud Escalation (Silent Test Passing):** Never remove `@unittest.skipIf` or wrapper exceptions only to replace them with silent `return` or `except: return` blocks. This destroys visibility into unsupported execution environments and falsely inflates the pass count (Tier 1 Blocker). Always use explicit `raise unittest.SkipTest(...)` for legitimately unsupported hardware paths or environments.

## Lessons Learned

### Architecture Quirks
- **.bss Section Obliteration Flaw**:: When using out-of-band IPC to stream host tensors before boot, explicitly declare main I/O arrays as file-scoped global arrays tagged with `__attribute__((section(".noinit")))` to prevent the C runtime `_start` sequence from zeroing them out.
- **AST Vectorization Explosions**:: Enforce strict upcasting limits (e.g., `max_upcast=28`) in custom renderers to preserve loop logic registers and prevent massive AST explosions that can cause compilation timeouts and OOM errors during downstream processing.
- **Environment Variable Race Conditions**:: Never mutate global states like `os.environ["PATH"]` directly in test files. This introduces fatal race conditions during `pytest -n auto` execution. Use `with unittest.mock.patch.dict(os.environ, ...)` instead.
- **Invalidation of Multi-GPU Assertions**:: Wrapping decorators like `needs_second_gpu` with blind exceptions that simply return `None` silently masks missing GPUs and flags tests as passed. Use `raise unittest.SkipTest("requires second gpu")`.
- **Renderer AST Customization**:: When mapping composite ML operations like Int8Linear (a `MUL` of two `CAST`ed `int8` inputs) to specialized target macros (e.g., `VDOT`), use the target-specific `extra_matcher` inside `CoralNPURenderer` to rewrite the UOp graph before C-style rendering. Defer FP16 scaling by ensuring `VDOT` outputs the `int32` accumulator type before applying the `CAST` to `float16`.

### Git & Environment Management
- **Illogical Task Ordering (Dependency Inversion)**:: Upstream syncs must be sequenced so dependencies (e.g., `tinygrad`) are synchronized before dependents that rely on them (e.g., `coralnpu`).
- **LIBCLANG_PATH Dependency Check**:: Always ensure `LIBCLANG_PATH` is explicitly set when invoking `pytest` for `tinygrad` tests, especially during upstream baseline validation, to avoid artificial `test_autogen.py` failures.
- **Path Coupling & Execution Failures**:: Correct path parameters in git commands to match the current working directory, avoiding relative path mismatches when executing inside submodules (e.g., do not run `git add tinygrad/...` when already inside `tinygrad`).

### Miscellaneous
- **Dependency Inversion in Testing**:: Re-sequence upstream syncs so `tinygrad` is synchronized before `coralnpu` when executing `coralnpu` tests.
- **Hardware Memory Abstraction Leak**:: Tinygrad VMM must enforce a strict `28KB` limit for the sum of all live `LOCAL` tensors (leaving 4KB for C-stack) and correctly manage bump allocator lifecycles to prevent DTCM OOM Memory Corruption.
- **Tinygrad DTCM Limit**:: `max_upcast` must be strictly capped at 28 to leave registers for loop logic, and the DTCM local tensor limit is 28KB (reserving 4KB for C-stack overhead).
- **VMM Memory Lifecycles (DTCM Tiling)**:: When defining local buffers (e.g. `DEFINE_LOCAL`) in custom renderers, do not use unbounded bump allocators that simply accumulate sizes. Instead, enforce strict VMM memory lifecycles by topologically analyzing the AST (transitively mapping UOp dependencies) to compute the true lifespan (start/end boundaries) of each local buffer, and use a first-fit linear scan allocator to recycle memory offsets and correctly cap the DTCM limit (e.g., 28KB peak).

### Testing Gotchas
- **Cross-Submodule Verification Integrity**:: When executing cross-submodule zero-trust baseline verification, `bazel test` MUST explicitly include `--override_repository` flags for ALL tracked ecosystem submodules (e.g., `mpact-riscv`, `mpact-sim`, `tinygrad`, `coralnpu-mpact`), not just the primary downstream dependency, to prevent stale local cache evaluation.
- **Dynamic Mock Base Calculation (IPC Tests)**:: When defining mock ELF structures in multiprocessing IPC tests (like `test_coralnpu_ipc.py`), strictly forbid hardcoded `self.mock_base` addresses (e.g., `0x80040000`). Dynamically compute the base address by summing the header and section lengths (e.g. `len(header + sh0...)`) combined with the `0x80000000` architectural offset and padding to ensure deterministic bounds.
- **FPGA DMA Verification**:: Tinygrad compilation must be verified against the bare-metal FPGA RTL (e.g. `coralnpu/fpga/`) to prove the double-buffering DMA scheduler avoids AXI deadlocks.
- **Fraudulent Happy-Path Padding (Fake Data)**:: Never use random Gaussian noise (e.g., Tensor.randn) in tests verifying mathematical bounds (e.g., test_real_world). Random noise masks accumulator overflows and zero-divisor faults. Replace with a deterministic, non-uniform authentic dataset (e.g., a scaled np.arange).
- **Integration Test Execution Boundary**:: Cross-submodule integration tests (e.g., executing tinygrad integrations via bazel) must be executed from the correct working directory (e.g., `/workspace/louhi_ws/coralnpu`) and properly format `--override_repository` flags.
- **JIT Hypothesis Deadline CI Scaling**:: `test_approx_jit_timeout` tests utilizing the `hypothesis` module to test JIT bounds must enforce a higher base CI timeout limit (e.g., `120000ms`) using `@settings(deadline=...)`, as multi-layer NPU caching can occasionally spike to 40 seconds on slow runner environments, causing flaky `DeadlineExceeded` failures.
- **Organic IPC Boundary Testing**:: When testing subprocess execution errors (e.g. `FileNotFoundError` or `CalledProcessError`), you must organically execute dummy shell scripts rather than using `unittest.mock.patch` on `subprocess` to artificially bypass the execution boundary.
- **Test Integrity in Backend Implementations**:: Handle unsupported device exceptions organically rather than relying on `@unittest.skipIf(Device.DEFAULT == "CUDA")`.
- **Test Invalidation (Multi-GPU)**:: In multi-GPU wrappers (like needs_second_gpu), do not silently return None on RuntimeError. Revert to raising unittest.SkipTest to correctly log a skip on single-GPU CI runners.
- **Test Skipping & Fraudulent Task Completion**:: Massive test skipping (e.g., bypassing 500+ tests via trivial `@unittest.skipIf` bounds or mock scripts) is a Tier 1 blocker that masks fundamental architectural failure surface areas. A 'passing' test suite does not guarantee validation if massive amounts of tests are skipped. Ensure all tests strictly execute natively and provide organic execution bounds (e.g., emitting dummy failing shell scripts) instead of bypassing execution via loophole rules.
- **Test Unmasking Cleanliness**: When deleting `@unittest.skipIf` or similar decorators to unmask tests, agents MUST ensure the resulting codebase remains structurally clean. Do not leave stray blank lines or abandon adjacent decorators. Use precise multi-line regex or explicit line-by-line replacement to completely eradicate the unmasking traces without breaking the adjacent formatting.
- **Testing Fraud Escalation (SkipTest)**:: Never remove explicit @unittest.skipIf decorators and replace them with silent 'return' or 'except: return' statements. This creates massive test masking. Always use 'raise unittest.SkipTest("Reason")' for legitimately unsupported hardware paths.
- **Tinygrad VMM Tiling Cap**:: `max_upcast` must be hard-capped at 28 (not 32) when `device == "CORALNPU"` to preserve architectural registers for loop logic.
- **Upstream Verify Timeout Scaling**:: When executing massive cross-repository verification builds (e.g., `coralnpu` with `tinygrad` and `coralnpu-mpact` overrides) inside sandboxes, Bazel executions can exceed 5-10 minutes. Agents must explicitly increase default CLI execution timeouts (e.g. to `1500000` ms) when running `bazel test //...` during Cycle Validation to prevent false-negative pipeline halts caused by standard shell timeouts.
- **VMM Dynamic Bounds Testing**:: VMM ELF parsing tests must dynamically compute boundaries (e.g., `0x80000000 + len(elf_hdr) + 0x2000`) instead of relying on hardcoded static payloads like `0x80004000` or explicit scalar baselines (`golden_bases = [0x80010000, ...]`). This mathematically proves the allocator dynamically extracts varying bounds. Ensure the underlying mocked ELF (e.g., `create_dummy_elf`) generates dynamic bounds accordingly.
- **Zero-Trust Verification Paths**:: When executing upstream baseline zero-trust verification via pytest inside a pristine worktree (like upstream/master), do NOT pass custom file paths (e.g. `test_coralnpu_renderer.py`) that only exist in the downstream local branch. This will cause pytest to fatally abort with error code 5 (No tests collected) during the collection phase. Only pass paths that structurally exist in the pristine upstream branch.
