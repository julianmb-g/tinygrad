# tinygrad Agent Instructions

## Lessons Learned

### Architecture Quirks
- **Architecture Validation (ABI)**:: Ensure mem_rmask and mem_wmask are uint8_t, not 32-bit, for a 64-bit payload.
- **Architecture Validation (DMA)**:: Mandate explicit DMA synchronization UOps (e.g., WAIT_DMA_READY) before the UOp graph transitions to the next compute chunk to prevent compute on stale data.
- **Buffer Mask Intersections & Global Memory**:: When tracking structural memory pointers (e.g., for DMA barrier consolidation), traverse the `u.src` tree recursively to locate base buffers. Note that Tinygrad does not use `Ops.DEFINE_GLOBAL`; you must check against `{Ops.DEFINE_LOCAL, Ops.DEFINE_REG, Ops.PARAM}`.
- **IPC Memory Mapping**:: When using `multiprocessing.shared_memory.SharedMemory`, assign `shm.buf` directly to memory views. Do not cast it via `ctypes` `from_buffer` to avoid unnecessary dependencies and type checking errors.

- **Late-Dequantization Execution**:: Deep W8A8 operations overflow standard 32-bit accumulators. Mid-loop chunked dequantization explodes memory limits and falsely assumes an FPU. Dequantization must be strictly deferred out-of-band to the Host CPU.
- **PyBind11 Out-of-Band Isolation**:: Native PyBind11 bindings inside the Python tuning loop cause crashes. Extract and wrap calls via `multiprocessing` workers using zero-copy shared memory.
- **SPSC Struct Sizing and ABI Conflicts**:: Hardware telemetry structs (like `rvvi_trace_event_t`) must rigorously align with memory models. A 32-bit `mem_rmask` for a 64-bit payload is an ABI contradiction and must be constrained to `uint8_t`.
- **Target-Aware DMA Synchronization Constraints**:: Do not unconditionally inject `WAIT_DMA_READY` full-pipeline barriers after `COPY` operations for every subsequent memory operation (`LOAD` or `STORE`). Implement a `buffer_mask` intersection tracker that traverses the `UOp` graph (`u.src`) to isolate the root buffers (`Ops.DEFINE_LOCAL`, `Ops.DEFINE_REG`, `Ops.PARAM`) and forces a barrier *only* if the accessed structural pointers overlap with the pending asynchronous `COPY`. Note: `Ops.DEFINE_GLOBAL` does not exist in Tinygrad; global arrays map to `Ops.PARAM`.

### Miscellaneous
- **DTCM Local Tensor Bounding**:: When optimizing tensors in `heuristic.py`, strictly enforce the DTCM Local Tensor Bounding exactly at 28KB. The `max_upcast` must be aligned to guarantee that tensors never exceed the 28KB hardware limit boundary (e.g., capping at 28).
- **Deferred Dynamic Register Spilling**:: Immediate `spill_load` provides zero peak-register pressure relief. Implement deferred dynamic spilling where `LOAD` is withheld until precise boundary consumption, clamping compiler limits (e.g., `max_upcast = 28`) to save pointer registers.
- **Fatal x86/RISC-V `ctypes` Host Loading**:: Do not attempt to use `ctypes.CDLL` to load cross-compiled `.so` shared libraries built for RISC-V (`-march=rv32imf_zve32x`) into the native x86 Python host process. This triggers `OSError: Exec format error`.
- **Full INT8 W8A8 Quantization Overflow**:: Standard 32-bit accumulators will mathematically overflow during large MAC loops before the deferred dequantization step occurs. You must implement intermediate scaling or widen the accumulator width.
- **RISC-V Native Execution Error**:: x86 Linux processes cannot natively execute RISC-V `ctypes`. The host-side `ctypes` bindings for RISC-V binaries must be severed and execution must route to an out-of-band hardware simulator.
- **Register Spilling (Belady's MIN)**:: When spilling registers due to active variable cap limits, ensure liveness tracking computes the *furthest* future usage rather than indiscriminately popping the oldest registered UOp. Spilling the oldest UOp often leads to immediate `spill_load` thrashing on the very next instruction, completely destroying register relief.
- **Register Spilling Liveness**:: When implementing deferred register spilling, `u.src` iterates over the original UOp graph. You must track and apply `replacements` mapping the original UOp to its newly `LOAD`ed active register representation to ensure topological references correctly map to the spill buffer instead of the original dead register.

- **Target-Aware Asynchronous Tracking**:: Blind `WAIT_DMA_READY` full-pipeline barriers destroy ping-pong schedules. Implement a target-aware asynchronous tracker (e.g., `buffer_mask` intersection) to synchronize overlapping structural pointers properly.

### Testing Gotchas
- **AST Erasure via Hardcoded Compiler Defects**:: Tests must mathematically assert the optimal AST representation to enforce compilation fidelity. Do not hardcode mathematically unsimplified, bloated AST representations just to achieve a passing test.
- **Autogen Clang Verification**:: When executing `pytest` across the `tinygrad` codebase, `LIBCLANG_PATH=/usr/lib/llvm-19/lib/libclang.so` must be explicitly exported. Otherwise, `autogen` tests will fail with an `AttributeError` failing to load `libclang.so`.
- **CStyleLanguage Context Initialization**:: When extracting dynamically generated attributes (like `dtcm_bump`) from the `CoralNPURenderer` context inside `CStyleLanguage` rewrites, always use `getattr(ctx, 'dtcm_bump', 4096)`. Manual `render_kernel` invocations in tests bypass the standard `render()` initialization, causing silent `AttributeError` failures if accessed directly.
- **Cosmetic Log Parsing (AST Validation)**:: When verifying compiler AST graphs or register spilling logic, do not rely on raw textual substring matching of debug C-comments (e.g., parsing `==== UOp Graph ====`). Tests must organically evaluate the structured `UOp` object graph or execute the compiled logic natively to prevent extreme fragility when debug formats change.
- **JIT Compilation Latency Regressions (Property Tests)**:: Property test timeout regressions (e.g. `DeadlineExceeded` in Hypothesis) MUST NOT be bypassed by stripping the latency bound (`deadline=None`). The explicit timeout bound must be preserved and carefully scaled (`120000ms`), or the underlying regression fixed, to prevent unbounded wait times masking infinite loops.
- **Late-Dequantization Test Parity**:: When eradicating mid-loop chunked dequantization, ensure that test suites explicitly verifying organic AST generation (e.g., `test_vdot_mapping`) are updated to remove hardcoded checks for intermediate operations like `int64_t` widening or `>>8ll` scaling, as these operations are definitively removed from the compilation pipeline.
- **Real World Memory Boundary Assertions**:: Adjust the `max_memory_allowed` threshold downwards to reflect new, more efficient memory usage baselines instead of arbitrarily asserting fixed bounds limits.
- **Structural AST Validation (Test Integrity)**:: Do not use cosmetic string matching (e.g., counting `WAIT_DMA_READY` or log parsing `==== UOp Graph ====`) to validate structural AST ordering, register spilling, or barrier placement in tests. Always intercept and organically evaluate the actual `UOp` objects or AST nodes, or natively execute the compiled payload to prove chronological constraints.
- **Structural Timeline vs String Counting (DMA Barriers)**:: When asserting the placement of synchronization barriers (like `WAIT_DMA_READY`), blind substring counts (e.g., `body.count("WAIT_DMA_READY") == 1`) are structurally meaningless and constitute test masking. Tests must explicitly evaluate the sequential ordering of the generated C-code or UOp AST to prove the barrier strictly occurs *after* the disjoint memory operations it is meant to synchronize.
- **Test Integrity (Deterministic Padding & Bounding)**:: When enforcing strict mathematical bounds checking on real-world networks (e.g. in `test_real_world.py`), ensure `derandomize_model()` explicitly populates weights with initialized, contiguous constants like `Tensor.full(...).contiguous()` rather than uninitialized `Tensor.empty(...)`, which yields non-deterministic `NaN`s and causes bounds verification logic to fail. Additionally, replace exponentially scaling input tensors (`Tensor.arange(...) * 0.1`) with modulus-bounded datasets (e.g. `(Tensor.arange(...) % 10) * 0.1`) to prevent FP16 mathematical overflows during convolution layers while maintaining dataset uniqueness.

- **Encapsulation & Proxy Test Evasion**:: When verifying structural compiler outputs (like UOp bounds or injected DMA macros), never use Python mock interceptors to bypass private class rendering methods (e.g., `unittest.mock.patch.object(renderer, "_render")`). Verification MUST be performed organically on the emitted graph, AST, or fully compiled binary to preserve integration fidelity.
- **Encapsulation Violation via Private Method Mocking**:: Private method interception for test coverage (e.g., `unittest.mock.patch.object(renderer, '_render')`) is strictly forbidden. Evaluate outputs organically.
- **Organic Target String Validation**:: When the framework produces a final target string (like C++ code) that cannot be intercepted without breaking encapsulation, tests must explicitly validate the generated string using structural assertions (e.g., grep for variables or macro calls and verify relative line occurrences) instead of patching internal rendering mechanisms.
- **Integer Exponentiation & Precision Loss**:: When fixing `RuntimeError` due to missing integer power support, do not blindly cast both bases and exponents to `float32` (which ruins precision for large integers like `11**7`). Explicitly cast them to `dtypes.double` (`float64`) before computing `UOp.pow` to preserve exact integer values, then truncate and cast back.
- **Fraudulent Test Masking via `assertRaises`**:: Do not replace failing equality assertions with `assertRaises(AssertionError)` simply because the codebase natively computes them incorrectly. This is a severe form of test evasion. If a mathematical result fails due to precision limits, fix the underlying dtype constraints or provide deterministic mathematically sound bounds.
- **Test Discovery Robustness**: When implementing global test discovery routines (e.g. iterating over files to filter tests), strictly avoid bare `except:` and `pass` blocks. Catch explicit `Exception` and log it to prevent silently masking file permission failures or corrupt test files.
- **Encapsulation & Proxy Test Evasion**:: When verifying structural compiler outputs (like UOp bounds or injected DMA macros), never use Python mock interceptors to bypass private class rendering methods (e.g., `unittest.mock.patch.object(renderer, "_render")`). Verification MUST be performed organically on the emitted graph, AST, or fully compiled binary to preserve integration fidelity.
- **Redundant Commit Bypassing**: When an agent discovers that a codebase modification task marked as pending in `PLAN.md` was already committed under a previous orchestration cycle's message, the agent MUST organically verify the tests pass and safely bypass creating redundant empty commits, explicitly checking off the tasks with a `(Verified: codebase modifications were previously committed...)` note to advance the stateful SDLC loop.
- **Isolated Phase 2 Pointer Verification**: Even if a submodule pointer was previously synchronized and committed out-of-band by an overlapping cycle, the Build Agent must still execute the cross-submodule zero-trust verification (e.g., `bazel test //...`) mapped in Phase 2 before officially checking off the task in `PLAN.md` to guarantee structural integrity before advancing the SDLC loop.
