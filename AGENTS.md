# tinygrad Agent Instructions

## Lessons Learned

### Architecture Quirks
- **Architecture Validation (DMA)**: Mandate explicit DMA synchronization UOps (e.g., WAIT_DMA_READY) before the UOp graph transitions to the next compute chunk to prevent compute on stale data.
- **Buffer Mask Intersections & Global Memory**: When tracking structural memory pointers (e.g., for DMA barrier consolidation), traverse the `u.src` tree recursively to locate base buffers. Note that Tinygrad does not use `Ops.DEFINE_GLOBAL`; you must check against `{Ops.DEFINE_LOCAL, Ops.DEFINE_REG, Ops.PARAM}`.
- **Late-Dequantization Execution**: Deep W8A8 operations overflow standard 32-bit accumulators. Mid-loop chunked dequantization explodes memory limits and falsely assumes an FPU. Dequantization must be strictly deferred out-of-band to the Host CPU.
- **Memory Abstraction**: Implement a Virtual Memory Management (VMM) abstraction or pseudo-driver layer in `ops_coralnpu.py` to handle ITCM/DTCM mapping internally, preventing hardware abstraction leaks.
- **Target-Aware DMA Synchronization Constraints**: Do not unconditionally inject `WAIT_DMA_READY` full-pipeline barriers after `COPY` operations for every subsequent memory operation (`LOAD` or `STORE`). Implement a `buffer_mask` intersection tracker that traverses the `UOp` graph (`u.src`) to isolate the root buffers (`Ops.DEFINE_LOCAL`, `Ops.DEFINE_REG`, `Ops.PARAM`) and forces a barrier *only* if the accessed structural pointers overlap with the pending asynchronous `COPY`. Note: `Ops.DEFINE_GLOBAL` does not exist in Tinygrad; global arrays map to `Ops.PARAM`.

### Git & Environment Management
- **Zero-Trust Baseline Path Coupling**: When executing `bazel test --override_repository` inside an out-of-tree isolated baseline worktree (e.g. `/tmp/coralnpu-baseline`), you MUST strictly enforce absolute paths (e.g. `--override_repository=tinygrad=/workspace/louhi_ws/tinygrad`). Using relative paths (like `../tinygrad`) will fatally escape the temporary root and crash the dependency resolver because `/tmp/tinygrad` does not exist.

### Miscellaneous
- **DTCM Local Tensor Bounding**: When optimizing tensors in `heuristic.py`, strictly enforce the DTCM Local Tensor Bounding exactly at 28KB. The `max_upcast` must be aligned to guarantee that tensors never exceed the 28KB hardware limit boundary (e.g., capping at 28).
- **Deferred Dynamic Register Spilling**: Immediate `spill_load` provides zero peak-register pressure relief. Implement deferred dynamic spilling where `LOAD` is withheld until precise boundary consumption, clamping compiler limits (e.g., `max_upcast = 28`) to save pointer registers.
- **Full INT8 W8A8 Quantization Overflow**: Standard 32-bit accumulators will mathematically overflow during large MAC loops before the deferred dequantization step occurs. You must implement intermediate scaling or widen the accumulator width.
- **Register Spilling (Belady's MIN)**: When spilling registers due to active variable cap limits, ensure liveness tracking computes the *furthest* future usage rather than indiscriminately popping the oldest registered UOp. Spilling the oldest UOp often leads to immediate `spill_load` thrashing on the very next instruction, completely destroying register relief.
- **Register Spilling Liveness**: When implementing deferred register spilling, `u.src` iterates over the original UOp graph. You must track and apply `replacements` mapping the original UOp to its newly `LOAD`ed active register representation to ensure topological references correctly map to the spill buffer instead of the original dead register.

### Testing Gotchas
- **Architectural Blueprint Refinement & Stateful Pruning**: When updating `DESIGN.md` based on `SPECS.md` and `REVIEW.md`, ensure that constraints like `max_upcast` vs DTCM physical limits are decoupled. Ensure OS-level `PR_SET_PDEATHSIG` and `atexit` bindings for persistent IPC pools to prevent "Node Down" zombie process deadlocks. When clearing resolved items from `REVIEW.md`, strictly apply block-based regular expressions to prevent catastrophic data loss of unrelated tracking states.
- **Authentic Mock Testing Data (Analytical Models)**: Tests evaluating mathematical cost estimators or probabilistic logic MUST use authentic, non-uniform deterministic datasets (like a scaled `np.arange`) rather than unrealistic fake arrays (like `np.ones * 0.1` or `np.eye(4)`) to prevent random value generation from causing `COST=0.0` or masking algorithmic failure.
- **Autogen Clang Verification**: When executing `pytest` across the `tinygrad` codebase, `LIBCLANG_PATH=/usr/lib/llvm-19/lib/libclang.so` must be explicitly exported. Otherwise, `autogen` tests will fail with an `AttributeError` failing to load `libclang.so`.
- **Cosmetic Log Parsing (AST Validation)**: When verifying compiler AST graphs or register spilling logic, do not rely on raw textual substring matching of debug C-comments (e.g., parsing `==== UOp Graph ====`). Tests must organically evaluate the structured `UOp` object graph or execute the compiled logic natively to prevent extreme fragility when debug formats change.
- **DMA Synchronization**: Tinygrad codegen MUST inject explicit DMA synchronization UOps (e.g., `WAIT_DMA_READY`) before transitioning to the next compute chunk to prevent vector MACs from computing on stale/partial fetches.
- **DTCM Decoupling & Max Upcast**: `max_upcast` must be decoupled from the raw 28KB DTCM bounds to allow `max_upcast=32` while ensuring `dtcm_peak` bounds tracking natively prevents overflows. Do not artificially clamp `max_upcast=28`.
- **Design Alignment (Architectural Debt)**: When structural constraints conflict with behavioral requirements (e.g., 28KB DTCM vs 32KB max_upcast), prioritize preserving hardware safety limits and document the deviation as architectural debt rather than silently overriding bounds.
- **Deterministic Data Padding Shapes**: When padding deterministic arrays for tests (e.g. `np.arange(X)`), ensure `X` accurately matches the total flattened size of the target `.reshape(N, M)` to prevent catastrophic `ValueError` shape evasion.
- **Encapsulation & Proxy Test Evasion**: When verifying structural compiler outputs (like UOp bounds or injected DMA macros), never use Python mock interceptors to bypass private class rendering methods (e.g., `unittest.mock.patch.object(renderer, "_render")`). Verification MUST be performed organically on the emitted graph, AST, or fully compiled binary to preserve integration fidelity.
- **Eradication of Global Exception Swallowing**: The `tinygrad` test suite was severely degraded by blanket `except (RuntimeError, Exception) as e: raise unittest.SkipTest(str(e))` wrappers, which mathematically eviscerated CI testing and masked catastrophic IPC simulator deadlocks. Agents must never introduce such wrappers; tests must fail organically. `unittest.SkipTest` should be exclusively reserved for strict hardware capability limitations (e.g., missing FP64 support).
- **Fraudulent "Happy-Path" Padding via Fake Data**: Tests verifying mathematical algorithms (like memory boundaries or cost models) MUST NOT be driven by random Gaussian noise (e.g., `Tensor.randn`). Random data completely masks algorithmic and mathematical failure surfaces (like accumulator overflows or zero-divisor faults) that are only exposed by deterministic, non-uniform authentic datasets. Always use deterministic datasets (e.g. a scaled `np.arange`).
- **Identical Operands Evasion**: Multiplying a matrix strictly by itself (A x A) generates fundamentally symmetric distributions. This mathematically masks layout transpositions and dimension swapping. Tests MUST use dimensional asymmetry (e.g., M != N != K) and deterministically scaled, non-uniform datasets (e.g., `np.arange`) to organically test kernel logic.
- **Integer Exponentiation & Precision Loss**: When fixing `RuntimeError` due to missing integer power support, do not blindly cast both bases and exponents to `float32` (which ruins precision for large integers like `11**7`). Explicitly cast them to `dtypes.double` (`float64`) before computing `UOp.pow` to preserve exact integer values, then truncate and cast back.
- **Memory Mapping & Loop Constraints**: The unroll boundary parameter (`max_upcast=32`) must be handled separately from the physical chunk boundary (28KB DTCM restriction).
- **Orphaned Task Ledger Updating**: When resuming an aborted execution cycle, explicitly check the git commit history of submodules to verify if the previous tasks were already completed before repeating redundant codebase modifications. \n- **Phase 5 Ledger Serialization Atomicity**:: When checking off Phase 5 Submodule Pointer Serialization in `PLAN.md`, the root workspace orchestrator commit MUST atomcially stage the updated `PLAN.md` concurrently with the submodules (e.g. `coralnpu`, `tinygrad`) to tightly bind the pointer state to the checked-off plan phase.
- **Private Method Mocking (Framework Creep)**: Bypassing public ML APIs (like `.pool()`) via private interceptors (like `._pool()`) and injecting unauthorized integer exponentiation hacks into third-party ML frameworks (like tinygrad) purely to pass local tests is strictly forbidden. Known framework limitations must be organically trapped via `assertRaises`.
- **Shape Evasion & Numpy Padding (Test Fraud)**: Array initializations must mathematically evaluate to the correct flattened target size (e.g., `np.arange(160)`) instead of using small sizes padded with zeros or modulo arithmetic that crash during reshaping.
- **Structural AST Validation (Test Integrity)**: Do not use cosmetic string matching (e.g., counting `WAIT_DMA_READY` or log parsing `==== UOp Graph ====`) to validate structural AST ordering, register spilling, or barrier placement in tests. Always intercept and organically evaluate the actual `UOp` objects or AST nodes, or natively execute the compiled payload to prove chronological constraints.
- **Structural Timeline vs String Counting (DMA Barriers)**: When asserting the placement of synchronization barriers (like `WAIT_DMA_READY`), blind substring counts (e.g., `body.count("WAIT_DMA_READY") == 1`) are structurally meaningless and constitute test masking. Tests must explicitly evaluate the sequential ordering of the generated C-code or UOp AST to prove the barrier strictly occurs *after* the disjoint memory operations it is meant to synchronize.
- **Test Integrity (Deterministic Non-Uniform Padding)**: When resolving testing fraud flagged by QA, completely replace trivial data sets like `np.eye` with non-uniform deterministically scaled datasets (e.g., `(np.arange(...) * 0.1).astype(np.float32)`) to ensure algorithmic limitations such as FP16 accumulator limits are actually tested, rather than mathematically masked by excessive zeros.
- **Test Integrity (Deterministic Padding Shapes via `math.prod`)**: When regenerating or refactoring deterministic arrays in tests with `np.arange`, never rely on placeholder shapes like `np.array([1]).shape` as arguments to `math.prod()`. `np.arange(math.prod(np.array([1]).shape))` evaluates to a size of 1, causing immediate `ValueError` crashes when attempting to `.reshape()` to the true target tensor dimensions. You MUST explicitly pass the correct target shape tuple directly to `math.prod()` (e.g. `math.prod((1, 1, 32, 32, 32))`) to ensure array size matches the layout constraints.
- **AXI Burst Boundary Static Assertion Fragility**: When switching AXI burst logic from statically chunked sizes to dynamic runtime boundary calculations (e.g., `4096 - (addr & 0xFFF)`), tests previously asserting explicit string bounds (like the hardcoded remainder `'904'`) will fail. Refactor the tests to assert the structural AST loops or generated condition variables instead of brittle runtime string remainders.
