# tinygrad Agent Instructions

## Lessons Learned

### Architecture Quirks
- **AST Vectorization Explosions:** Enforce strict upcasting limits (e.g., `max_upcast=28`) in custom renderers to preserve loop logic registers and prevent massive AST explosions that can cause compilation timeouts and OOM errors during downstream processing.
- **Analytical Model Trivialization:** When testing cost estimators (e.g., in `test_coralnpu_renderer.py`), tests must use a deterministic, valid `_cost_model` dictionary that tests the actual numerical bounds (like `0.1` softplus activation bounds) rather than trivializing the test by stripping parameters and asserting `cost >= 0.0` with a static 0.0.
- **.bss Section Obliteration Flaw:** When using out-of-band IPC to stream host tensors before boot, explicitly declare main I/O arrays as file-scoped global arrays tagged with `__attribute__((section(".noinit")))` to prevent the C runtime `_start` sequence from zeroing them out.
- **DTCM Tiling Limits & Double-Buffering Overflows:** Always enforce the 12KB DTCM limit in the renderer by explicitly verifying the sum of `Ops.DEFINE_LOCAL` buffer sizes. When tiling loops, chunks must be strictly scaled to accommodate stack overhead (e.g., <= 12KB chunks, not 16KB). The limit applies to the sum of all live `LOCAL` tensors to prevent runtime memory overflow, and lower DMA via `Ops.COPY` to asynchronous hardware macros like `CORAL_DMA_ASYNC`.
- **Renderer AST Customization:** When mapping composite ML operations like Int8Linear (a `MUL` of two `CAST`ed `int8` inputs) to specialized target macros (e.g., `VDOT`), use the target-specific `extra_matcher` inside `CoralNPURenderer` to rewrite the UOp graph before C-style rendering. Defer FP16 scaling by ensuring `VDOT` outputs the `int32` accumulator type before applying the `CAST` to `float16`.

### JIT Test and VMM Hardcoding Flaws
- **Analytical Mock Authentic Data:** Tests evaluating cost estimators MUST use authentic, non-uniform deterministic arrays (like `np.arange` normalized) instead of trivial `np.ones * 0.1` padding. This prevents fraudulent bounds testing (like `cost > 0.0` always passing or randomly failing due to `random.gauss` when standard deviations blow up).
- **Analytical Model Testing & Determinism**: When asserting against stochastic estimator functions (e.g. `estimate_cost` using `random.gauss`), you must explicitly seed the random number generator (e.g., `random.seed(0)`) within the test function to ensure the sampled values are deterministic, permitting reliable bounds testing and preventing flaky AssertionError failures.
- **JIT Hypothesis Deadline CI Scaling:** `test_approx_jit_timeout` tests utilizing the `hypothesis` module to test JIT bounds must enforce a higher base CI timeout limit (e.g., `120000ms`) using `@settings(deadline=...)`, as multi-layer NPU caching can occasionally spike to 40 seconds on slow runner environments, causing flaky `DeadlineExceeded` failures.
- **VMM Dynamic Boundary Parsing Mock:** When writing tests that mock `CORALNPU_ELF` parsing for memory allocators, DO NOT hardcode `.bss` boundaries like `0x80004000`. Instead, dynamically calculate the payload offset (e.g. `0x80000000 + len(elf_hdr) + 0x2000`) and embed it in the mocked `_end` symbol table to prove the allocator strictly honors the parsing bounds rather than relying on an artificial assumption.
- **VMM Dynamic Bounds Testing:** VMM ELF parsing tests must dynamically compute boundaries without relying on hardcoded static payloads like 0x80004000 to prove accurate extractions.
- **VMM ELF Relocation Fraud:** When patching ELF dynamic boundary parsing tests, ensure the underlying mocked ELF (e.g., `create_dummy_elf`) is also updated to generate dynamic bounds instead of merely patching the caller to use a dynamic variable against a hardcoded artificial baseline like `0x80004000`.

\n- **Zero-Trust Verification Paths:** When executing upstream baseline zero-trust verification via pytest inside a pristine worktree (like upstream/master), do NOT pass custom file paths (e.g. test_coralnpu_renderer.py) that only exist in the downstream local branch. This will cause pytest to fatally abort with error code 5 (No tests collected) during the collection phase. Only pass paths that structurally exist in the pristine upstream branch.
- **VMM Dynamic Boundary Parsing Fraud:** When implementing dynamic VMM ELF parsing tests, do not rely on hardcoded explicit scalar baselines (e.g., `golden_bases = [0x80010000, ...]`) to verify allocations. Tests must dynamically locate the `_end` symbol and mathematically prove the allocator extracts varying bounds dynamically, rather than bypassing testing via hardcoded arrays.
- **VMM Bump Allocator Memory Leak:** Linear accumulation without reuse guarantees OOM on sequential executions. Update the VMM spec to implement true memory lifecycle (e.g., arena resets per-inference or tensor address ref-counting).
